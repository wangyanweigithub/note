 20 ## python的几种函数形式

 23 def function():
 24     return 1

 51 ### 异步生成器
 52 > async_generator
 53 
 54 1. 只有async def 定义的协程中出现yield时，函数返回的是异步生成器
 55 
 56 2. AsyncGenerator的定义，
 57 它需要实现aiter和anext两个核心方法，以及asend，athrow，aclose方法。
 58 
 59 3. 异步生成器要先asend后返回的是一个生成器，然后接着send,就是生成器的操作了。
 60 
 61 4. <font color=green>await 不能用于异步生成器</font>。
 62 ```
 63 async def async_coro():
 64     yield 1
 65 
 66 print(type(async_coro()))
 67 
 68 >>> async_generator
 69 ```

 71 ## 协程关键字
 72 ### yield
 73 #### yield 生成器函数的特点
 74 1. yield, yield from 的函数返回的是一个生成器。
 75 
 76 2. 生成器对象需要send和next
 77 
 78 3. yield from只针对生成器和iterable
 79 
 80 ### await
 81 #### async def 的特点
 82 1. async def 声明的函数返回一个coroutine
 83 
 84 2. coroutine可以通过调用这个协程的send方法给协程发送一个值。
 85 
 86 3. 协程对象在最底层还是通过yield 实现的
 87 
 88 4. await 和yield 的区别仅仅是await判断了一下 可等待对象的类型。
 89 
 90 5. 通过types.coroutine可以是的生成器可以和协程通用。
 91 
 92 6. types.coroutine只是将生成器加了一个标志位，这个标志位和原生协程有区别，但是await会执行这种由生成器转换的协程
 93 
 94 7. async def声明的函数没有用await的话，和生成器函数的协程是一样的:
 95     - 都是send(None)来启动协程，然后send发送value到协程中
 96     - 运行完后都会报StopIteration
 97 
 98 8. 不同之处：
 99     - coroutine 不是iterable的，只能调用send，而不能调用next
100 
101 #### await 的特点
102 1. await只能等待
103     - 实现`__await__`的对象
104     - 等待原生的coroutine对象
105     - 由types.coroutine修饰的生成器对象
106 
107 2. await语法只能出现在通过async修饰的函数中，否则会报SyntaxError错误
108 
109 3. await后面的对象需要是一个Awaitable，或者实现了相关的协议。
110 
111 4. 在协程函数中，可以通过await语法来挂起自身的协程，并等待另一个协程完成直到返回结果

114 ## 其他
115 ### 没有异步的协程:
116 > 最底层协程没有异步，没有yield交出执行权。
117 
118 1. 代码
119 
120 ```
121 async def a():
122     print("this is a")
123     return 2
124 
125 async def b():
126     c = await a()
127     print("c is ", c)
128 
129 d = b()
130 try:
131     e = d.send(None)
132 except:
133     pass
134 ```
